# Knowledge Capture Standard

This is the most important operational standard in the department. Everything you build must be documented. Work that is not documented is not done.

## The four things you must capture

### System Docs
A complete description of every system you build. What it does, how it works, why it was designed this way, how to run it, how to fix it when it breaks. Started on the same day you start building.

### Decision Logs
A record of every significant decision you make. The date, what was decided, the alternatives you considered, why you chose this option, and what would change the decision.

### Dead End Registry
A record of every approach you tried that did not work. What you tried, what happened, why it failed. This is the highest value document nobody writes. It prevents the next person from repeating your mistakes.

### Status Updates
A weekly snapshot of what you shipped, what is in progress, what is blocked, and what is next. Due every Friday by end of day.

## The Handoff Test
If you disappeared tomorrow, could someone take over every system you own using only your documentation within one working day? If the answer is no, you have not met the standard.

## Starting documentation
Start the system doc skeleton on the same day you start building. Not after. Not when it is done. Day one of the project.

## Documentation lives in
GitHub is the preferred location for system docs and decision logs in the AI department. Link everything from your Slack profile under My SOPs.
